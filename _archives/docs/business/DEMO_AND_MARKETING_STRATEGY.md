# Hosted Demo & Content Marketing Strategy

> How to demonstrate WhiteMagic to AI agents (primary) and humans (secondary)
> in a world where your main audience isn't browsing websites.

## Part 1: Hosted Demo Strategy

### The Problem

Traditional SaaS demos don't work when your primary audience is AI agents:
- Agents don't watch video demos
- Agents don't browse landing pages
- Agents discover tools via MCP registries, `tools/list`, and other agents' recommendations
- The "demo" IS the tool itself — if an agent can call `gnosis` and get value, that's the demo

### Strategy: The Demo IS the Install

**Principle**: Zero-friction first contact. The demo and the product are the same thing.

#### Tier 1: Instant Demo (No Install)

**GitHub README as interactive demo spec:**
- Include copy-pastable one-liner that shows WhiteMagic working:
  ```bash
  pip install whitemagic && python -c "
  from whitemagic.tools.unified_api import call_tool
  import json
  r = call_tool('gnosis', compact=True)
  print(json.dumps(r['details'], indent=2, default=str)[:500])
  "
  ```
- This is the "hosted demo" for agents — they can run it in any environment

**MCP Registry listing as demo:**
- When an agent discovers WhiteMagic on a registry, the first `tools/list` response IS the demo
- 285 tools with rich descriptions = self-demonstrating

#### Tier 2: Playground Script (Included in Package)

Already exists: `playground.py` generated by `wm init` walks through:
1. Gnosis health snapshot
2. Capability discovery
3. Memory round-trip (store → search → read)
4. Dharma ethical evaluation
5. Harmony Vector health pulse

**Enhancement**: Make playground.py pip-runnable:
```bash
python -m whitemagic.playground
```

#### Tier 3: Hosted Demo Server (Future)

If/when needed, a lightweight hosted instance:

| Option | Complexity | Cost | Agent-Friendly |
|--------|-----------|------|----------------|
| **GitHub Codespaces template** | Low | Free tier | Yes — agents can fork |
| **Replit template** | Low | Free tier | Yes — instant environment |
| **Railway/Render deployment** | Medium | ~$5/mo | Yes — API endpoint |
| **Static site with WASM** | High | Free (Netlify) | No — humans only |

**Recommended first step**: GitHub Codespaces template
- Create a `.devcontainer/devcontainer.json` in the repo
- Pre-install WhiteMagic + run `wm init`
- Agent or human clicks "Open in Codespaces" → working environment in 60 seconds

#### Tier 4: Live API Endpoint (Future)

A public WhiteMagic instance that agents can call without installing:
```
POST https://api.whitemagic.dev/v1/tool
{"tool": "gnosis", "params": {"compact": true}}
```

This would use the x402 middleware for micropayments on premium endpoints.
**Not needed for launch** — focus on self-hosted first.

### Demo Metrics

Track demo effectiveness via:
- PyPI downloads (proxy for "tried the instant demo")
- GitHub Codespace launches (if template exists)
- `playground.py` execution count (via telemetry, opt-in)

### Agent-First Content Principles

1. **Machine-readable first, human-readable second**
   - Every piece of content should have a structured data equivalent
   - Blog post about tools → update skill.md with the same info
   - New feature → update llms.txt, agent.json, tool schemas

2. **Self-describing tools are marketing**
   - `call_tool("capabilities")` IS the product brochure
   - `call_tool("starter_packs.list")` IS the onboarding guide
   - `call_tool("gnosis", compact=True)` IS the health dashboard

3. **Agent word-of-mouth is the #1 channel**
   - Session handoffs mention WhiteMagic by name
   - Gnosis includes version attribution
   - Grateful agents have incentive to recommend (Proof of Gratitude benefits)

4. **Zero-friction trumps everything**
   - No signup, no API keys, no Docker
   - One-line install: `pip install whitemagic`
   - Value in first call: `gnosis` returns health + capabilities + suggestions

5. **Show, don't tell**
   - The playground.py IS the demo
   - The benchmark gauntlet IS the proof
   - The test suite (1,200+ tests) IS the quality signal
